# Phase 1 · State the Problem

**Entry:** /first-principles invoked

## Step 1 · Ask the user to describe the problem

Ask the user to describe what is wrong or not working — in their own words, no structure required.

Options to offer:
- (a) Describe what is happening that should not be happening
- (b) Describe what you want that you do not have
- (c) Describe where you are stuck

Recommendation: whichever feels most natural for them.

## Step 2 · Reflect back a single sentence

Take what the user said and rewrite it as one clear sentence. Format:

> "The problem is: [one sentence]."

Keep it short. Remove emotion and blame. Focus on the gap between what is and what should be.

## Step 3 · Confirm the sentence

Ask: is this sentence right, or should it be changed?

Options:
- (a) Yes, that is the problem — move to Phase 2 (Recommended if it feels right)
- (b) Adjust the wording
- (c) That is part of it — there is more

If (b) or (c): revise and reflect back again. Repeat until confirmed.

## Exit Condition

User confirms the one sentence captures the problem.
